
HOW TO ANALYSE NETWORK TRAFFIC
===================================================

Network traffic analysis is the process of monitoring
and examining data packets that move through a network
to ensure proper performance and security.

-------------------------------
USES OF NETWORK TRAFFIC ANALYSIS
-------------------------------
| S/N | Use                                                 |
|-----|-----------------------------------------------------|
| 1   | Detect security threats                             |
| 2   | Monitor network performance                        |
| 3   | Troubleshoot network problems                      |
| 4   | Identify heavy users and applications              |
| 5   | Prevent network congestion                         |

-------------------------------
NETWORK TRAFFIC ANALYSIS TOOLS
-------------------------------
| S/N | Tool Name                                          |
|-----|-----------------------------------------------------|
| 1   | Wireshark                                          |
| 2   | tcpdump                                            |
| 3   | ntopng                                             |
| 4   | iftop                                              |
| 5   | SolarWinds                                        |



| S/N | Step                                                |
|-----|-----------------------------------------------------|
| 1   | Define the purpose of analysis                     |
| 2   | Select a suitable network tool                    |
| 3   | Capture network traffic                           |
| 4   | Apply filters to focus on key data                |
| 5   | Analyse captured packets                          |

-------------------------------
COMMON NETWORK ISSUES
-------------------------------
| S/N | Issue                                               |
|-----|-----------------------------------------------------|
| 1   | Network congestion                                 |
| 2   | Excessive bandwidth usage                          |
| 3   | Dropped connections                                |
| 4   | Unauthorized access                                |

-------------------------------
BEST PRACTICES
-------------------------------
| S/N | Practice                                            |
|-----|-----------------------------------------------------|
| 1   | Monitor traffic regularly                          |
| 2   | Use filters to reduce unnecessary data             |
| 3   | Save and review network logs                       |
| 4   | Secure network access                              |
| 5   | Follow privacy and legal guidelines                |
